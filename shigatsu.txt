## 1. PROJECT OVERVIEW

### Purpose
Interactive gamified training platform for ISO Management Systems (IMS) including ISO 9001, 14001, and 45001 standards.

### Key Features
-  **Training Topics** with mandatory and bonus questions
-  **4 Game Types**: Text Twist, Wordle, Quick Quiz, Hangman
-  **XP & Leveling System** (100 XP = 1 level)
-  **Badge System** with yearly reset and re-earning
-  **Leaderboard** with rankings
-  **Admin Panel** for content management
-  **Training Logs** for completion history
-  **Activity Logs** for user actions
-  **User Profiles** with progress tracking

### User Roles
- **Admin**: Full access, content management, user management
- **User**: Training access, games, profile viewing

## 2. SYSTEM ARCHITECTURE

### Architecture Type
**3-Tier Architecture (MVC Pattern)**

```
┌─────────────────────────────────────────────┐
│           CLIENT (React.js)                 │
│  - UI Components                            │
│  - State Management                         │
│  - Routing                                  │
└─────────────────┬───────────────────────────┘
                  │
                  │ HTTPS/REST API
                  │
┌─────────────────▼───────────────────────────┐
│           SERVER (Node.js/Express)          │
│  - API Routes                               │
│  - Business Logic                           │
│  - Authentication/Authorization             │
│  - Data Validation                          │
└─────────────────┬───────────────────────────┘
                  │
                  │ Mongoose ODM
                  │
┌─────────────────▼───────────────────────────┐
│           DATABASE (MongoDB)                │
│  - Users Collection                         │
│  - Topics Collection                        │
│  - Games Collection                         │
│  - Badges Collection                        │
│  - Training Logs Collection                 │
│  - Activity Logs Collection                 │
└─────────────────────────────────────────────┘
```

### Data Flow

**User Login Flow:**
```
User → Login Form → POST /api/auth/login
→ Server validates credentials
→ Generates JWT token
→ Returns token + user data
→ Client stores token in localStorage
→ Client redirects to dashboard
```

**Training Completion Flow:**
```
User completes topic
→ POST /api/topics/:id/submit-answers
→ Server validates answers
→ Awards XP and badge
→ Creates training log entry
→ Returns updated user data
→ Client updates UI
```

## 4. DATABASE SCHEMA

### 4.1 USER MODEL

```javascript
{
  _id: ObjectId,
  username: String (required, unique),
  email: String (required, unique),
  password: String (required, hashed),
  role: String (enum: ['user', 'admin'], default: 'user'),
  xp: Number (default: 0),
  level: Number (default: 1),
  
  completedTopics: [{
    topicId: ObjectId (ref: 'Topic'),
    mandatoryCompleted: Boolean,
    bonusCompleted: Boolean,
    bonusCorrect: Number,
    completedAt: Date
  }],
  
  badges: [{
    topicId: ObjectId (ref: 'Topic'),
    badgeName: String,
    badgeImage: String,
    badgeCount: Number (default: 1),  // Increments on re-earn
    earnedAt: Date
  }],
  
  createdAt: Date (default: Date.now)
}

// Indexes:
- username: unique
- email: unique
- xp: 1 (for leaderboard queries)
```

### 4.2 TOPIC MODEL

```javascript
{
  _id: ObjectId,
  title: String (required),
  description: String,
  content: String (required, HTML supported),
  category: String,
  difficulty: String (enum: ['easy', 'medium', 'hard']),
  badgeName: String (required),
  badgeImage: String (required),
  
  questions: [{
    question: String (required),
    options: [String] (4 options),
    correctAnswer: Number (0-3),
    isMandatory: Boolean (default: false)
  }],
  
  isActive: Boolean (default: true),
  order: Number (for sorting),
  createdAt: Date (default: Date.now),
  updatedAt: Date
}

// Constraints:
- 1 mandatory question per topic
- 4 bonus questions per topic
- Questions array has exactly 5 questions

// Indexes:
- isActive: 1
- order: 1
```

### 4.3 GAME MODEL

```javascript
{
  _id: ObjectId,
  title: String (required),
  description: String (required),
  gameType: String (enum: ['texttwist', 'wordle', 'quickquiz', 'hangman'], required),
  difficulty: String (enum: ['easy', 'medium', 'hard'], required),
  maxXP: Number (required, default: 100),
  timeLimit: Number (seconds, 0 = no limit),
  isActive: Boolean (default: true),
  
  content: Mixed {
    // Text Twist
    words: [{
      mainWord: String,
      subwords: [String]
    }],
    
    // Wordle
    words: [{
      word: String (5 letters),
      hint: String
    }],
    
    // Quick Quiz
    questions: [{
      question: String,
      options: [String] (4 options),
      correctAnswer: Number (0-3),
      points: Number (default: 10)
    }],
    
    // Hangman
    words: [{
      word: String (phrase allowed),
      hint: String,
      category: String
    }]
  },
  
  completions: [{
    userId: ObjectId (ref: 'User'),
    score: Number,
    timeSpent: Number (seconds),
    completedAt: Date (default: Date.now)
  }],
  
  createdAt: Date (default: Date.now)
}

// Virtual Property:
completionCount: completions.length

// Methods:
hasUserCompleted(userId): Boolean
addCompletion(userId, score, timeSpent): Promise

// Indexes:
- gameType: 1
- isActive: 1
```

### 4.4 BADGE MODEL

```javascript
{
  _id: ObjectId,
  name: String (required),
  description: String,
  imageUrl: String,
  topicId: ObjectId (ref: 'Topic', required),
  year: Number (required),  // For yearly tracking
  badgeCount: Number (default: 0),  // Total users who earned this badge this year
  isActive: Boolean (default: true),
  createdAt: Date (default: Date.now)
}

// Compound Index:
- topicId + year: unique
```

### 4.5 TRAINING LOG MODEL

```javascript
{
  _id: ObjectId,
  userId: ObjectId (ref: 'User', required),
  topicId: ObjectId (ref: 'Topic', required),
  topicTitle: String (required),
  xpEarned: Number (required),
  mandatoryCompleted: Boolean (default: false),
  bonusCompleted: Boolean (default: false),
  badgeEarned: Boolean (default: false),
  completedAt: Date (default: Date.now)
}

// Indexes:
- userId + completedAt: -1 (for user history queries)
```

### 4.6 ACTIVITY LOG MODEL

```javascript
{
  _id: ObjectId,
  userId: ObjectId (ref: 'User', required),
  action: String (required),
  details: Mixed (optional),
  ipAddress: String,
  userAgent: String,
  timestamp: Date (default: Date.now)
}

// Indexes:
- userId + timestamp: -1
- action: 1
```

## 5. API ENDPOINTS

### 5.1 AUTHENTICATION ENDPOINTS

#### POST /api/auth/register
**Purpose**: Register new user

**Request Body**:
```json
{
  "username": "string (3-30 chars)",
  "email": "string (valid email)",
  "password": "string (min 6 chars)"
}
```

**Response (201)**:
```json
{
  "token": "jwt_token_string",
  "user": {
    "_id": "user_id",
    "username": "username",
    "email": "email",
    "role": "user",
    "xp": 0,
    "level": 1
  }
}
```

**Errors**:
- 400: Validation error (missing fields, invalid format)
- 409: User already exists

---

#### POST /api/auth/login
**Purpose**: User login

**Request Body**:
```json
{
  "email": "string",
  "password": "string"
}
```

**Response (200)**:
```json
{
  "token": "jwt_token_string",
  "user": {
    "_id": "user_id",
    "username": "username",
    "email": "email",
    "role": "user|admin",
    "xp": 100,
    "level": 2,
    "badges": []
  }
}
```

**Errors**:
- 400: Invalid credentials
- 404: User not found

---

#### GET /api/auth/user
**Purpose**: Get current user profile

**Headers**: 
```
Authorization: Bearer <token>
```

**Response (200)**:
```json
{
  "_id": "user_id",
  "username": "username",
  "email": "email",
  "role": "user",
  "xp": 250,
  "level": 3,
  "completedTopics": [...],
  "badges": [...]
}
```

**Errors**:
- 401: Unauthorized (no token or invalid token)

## 6. AUTHENTICATION FLOW

### JWT Token Structure

**Payload**:
```javascript
{
  _id: "user_id",
  role: "user|admin",
  iat: 1234567890,  // Issued at
  exp: 1234657890   // Expires (24 hours later)
}
```

**Token Storage**: LocalStorage (`key: "token"`)

**Header Format**:
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### Middleware: authenticateToken

```javascript
// Validates JWT token
// Adds req.user with { _id, role }
// Returns 401 if invalid
```

### Middleware: isAdmin

```javascript
// Checks if req.user.role === 'admin'
// Returns 403 if not admin
// Must come after authenticateToken
```

### Protected Route Pattern

**Backend**:
```javascript
router.get('/protected', authenticateToken, (req, res) => {
  // req.user is available
});

router.post('/admin-only', authenticateToken, isAdmin, (req, res) => {
  // Only admins reach here
});
```

**Frontend**:
```javascript
// ProtectedRoute component checks localStorage token
// Redirects to login if no token
// ProtectedRoute with adminOnly prop checks user.role
```

## 9. SETUP INSTRUCTIONS

### Prerequisites
- Node.js 16+ installed
- MongoDB Atlas account (or local MongoDB)
- Git installed
- Code editor (VS Code recommended)

### Step 1: Clone Repository

```bash
git clone https://github.com/YOUR_USERNAME/gamified-ims.git
cd gamified-ims
```

### Step 2: Setup Backend

```bash
cd server
npm install

# Create .env file
cp .env.example .env

# Edit .env with your values:
# MONGODB_URI=mongodb+srv://...
# JWT_SECRET=your_secret_key
# PORT=5000
```

### Step 3: Setup Frontend

```bash
cd ../client
npm install

# Create .env file
cp .env.example .env

# Edit .env:
# REACT_APP_API_URL=http://localhost:5000/api
```

### Step 4: Seed Database (Optional)

```bash
cd ../server
node seedGames.js  # Seeds 8 sample games
```

### Step 5: Run Development Servers

**Terminal 1 - Backend:**
```bash
cd server
npm run dev
```

**Terminal 2 - Frontend:**
```bash
cd client
npm start
```

### Step 6: Create Admin Account

Navigate to http://localhost:3000/register and create an account.

Then manually update in MongoDB:
```javascript
db.users.updateOne(
  { email: "admin@example.com" },
  { $set: { role: "admin" } }
)
```

## 11. SECURITY CONSIDERATIONS

- JWT tokens with 24-hour expiration

 **Authorization**:
- Role-based access control (user/admin)
- Admin-only routes protected

 **Input Validation**:
- Required fields validated
- Email format validation
- Password length requirements

 **Environment Variables**:
- Sensitive data in .env (not in git)

## 13. TROUBLESHOOTING

### Common Issues

**Issue**: "MongooseError: Operation buffering timed out"
**Solution**: Check MongoDB connection string, verify IP whitelist

**Issue**: "CORS Error"
**Solution**: Verify CLIENT_URL in backend .env matches frontend URL

**Issue**: "401 Unauthorized"
**Solution**: Check token in localStorage, verify token hasn't expired

**Issue**: "Cannot find module"
**Solution**: Run `npm install` in both client and server

**Issue**: "Port 5000 already in use"
**Solution**: Kill process on port 5000 or change PORT in .env

## APPENDIX A: XP & LEVELING FORMULA

```javascript
XP per Level = 100
Level = floor(Total XP / 100) + 1

Examples:
0-99 XP = Level 1
100-199 XP = Level 2
200-299 XP = Level 3
1000-1099 XP = Level 11
```
